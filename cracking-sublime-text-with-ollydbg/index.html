Cracking Sublime Text with Ollydbg

I've been using Sublime Text 3 (which I purchased) for a while and I was wondering if I could crack it with my basic knowledge.

My goal here is was to get rid of these:


The first window is the about window and the second is the nag screen. The nag appears if you hit ctrl+s (after a change was made) five times. After that I suppose there is a timer or a character counter because it won't pop for a while. Btw I'm using Sublime Text 3 build 6065 x86 for this demo. Let's get started. Fire Ollydbg and attach the process to it.
I've tried to search for strings like 'Unregistred' and 'Hello' right away but didn't find any. So I opened the about window, paused the program and hit 'K' to check the call stack.


Now right click on one of the procedures -> show call. Here we are, now we can look for strings (right click, search for -> all referenced string). I looked for 'Hello' and followed in disassembly. Here is the first badboy:


I looked arround the instructions and at 00C537A2 I notided this JNZ 00C53839 the address is pointing to a LEAVE instruction after the nag screen. So I just changed it for a JMP so we pass over it


Now the second bad boy. Search for the string "Unregistered" alone and follow in disassembly. From there I put a breakpoint at 00C7503A and followed the execution after I open the about window. I noticed that the JE at 00C7506D was jumping to the unregistered part. This is because of the CMP instruction before. The result is true and the Z flag is set to 1 .


So here you can either set the Z flag to 0 by clicking on it, or change 0 to 1 or any value other than 0 in the CMP instruction to continue to the registered part.

Voilà
